#嵌入式 
输出比较是一个相当重要的功能，它用于输出**PWM波形**，是驱动电机的必要条件

>[!danger] 非常注意
>在配置好**输出比较 OC**或者**输入捕获 IC**后，**定时中断**是依然可以实现的
>只是**编码器**需要占用时基电路的CNT，所以定时中断干不了了

# 输出比较 OC Output Compare
输出比较通过比较CNT与CCR寄存器值得关系，**来对输出电平进行置1、置0得翻转得操作**，用于输出**一定频率**和**占空比**得PWM波形，对，就是你想的那样。又对！这个PWM也是你想的那样！

# 脉冲宽度调制 PWM Pulse Width Modulation
通过输出一定频率，一定占空比的方波电流，用**数字信号**来模拟**模拟信号**
就打个比方，一个周期内我一半输出高电平，一半输出低电平，那么就是输出${1\over2}V_{cc}$

# 输出比较模式
|                                     |                                                                                          |
| ----------------------------------- | ---------------------------------------------------------------------------------------- |
| 模式                                  | 描述                                                                                       |
| 冻结 Timing                           | CNT=CCR时，REF保持为原状态                                                                       |
| 匹配时置有效电平 Active                     | CNT=CCR时，REF置有效电平                                                                        |
| 匹配时置无效电平 Inactive                   | CNT=CCR时，REF置无效电平                                                                        |
| 匹配时电平翻转 Toggle                      | CNT=CCR时，REF电平翻转                                                                         |
| 强制为无效电平 ( 低电平 ) ForcedAction_Active | CNT与CCR无效，REF强制为无效电平                                                                     |
| 强制为有效电平 ( 高电平 )ForcedAction_Inctive | CNT与CCR无效，REF强制为有效电平                                                                     |
| PWM模式1 PWM1                         | 向上计数：CNT<CCR时，REF置有效电平，CNT≥CCR时，REF置无效电平<br><br>向下计数：CNT>CCR时，REF置无效电平，CNT≤CCR时，REF置有效电平 |
| PWM模式2 PWM2                         | 向上计数：CNT<CCR时，REF置无效电平，CNT≥CCR时，REF置有效电平<br><br>向下计数：CNT>CCR时，REF置有效电平，CNT≤CCR时，REF置无效电平 |

显然，**PWM1**和**PWM2**模式的区别在于**极性(Polarity)不同**

# 输出比较的配置
**先来介绍一下相关库函数**

```C
TIM_OCxInit(TIMx , TIM_OCInitStruct); //初始化
TIM_OCStructInit(TIM_OCInitStruct); //结构体初始化

//下面是单独修改某些配置的
TIM_OCxPreloadConfig(TIMx , TIM_OCPreload_value); //重新写入CCR的值，更新事件后才生效

TIM_OCxPolarityConfig(TIMx , TIM_OCPolarity); //主通道的极性配置
TIM_OCxNPolarityConfig(TIMx , TIM_OCPolarity); //高级计时器中互补通道的极性配置

TIM_CCxCmd(TIMx , TIM_Channel , TIM_CCx); //这里的x就是x，不是数字，这是用来修改输出使能参数的
TIM_CCxNCmd(TIMx , TIM_Channel , TIM_CCxN);

TIM_SelectOCxM(TIMx , TIM_Channel , TIM_OCMode); //输出比较模式

TIM_SetComparex(TIMx , Comparex_value); //修改CCR寄存器的函数

TIM_CtrilPWMOutputs(TIMx , NewState); //高级定时器输出PWM时需要用这个使能主输出
```

同时需要注意，不同的比较口对应的输出**GPIO通道**不一样，**OC1**对应**AP0**，以此类推
**快速过一遍TIM的初始化，NVIC无需配置**

```C
AutoInitGPIO(...) //我们这里初始GPIOA的PA0引脚 
//但是要注意，我们模式应该用复用推挽输出，因为信号不来自CPU，而是片上外设，即TIM，GPIO_Mode_AF_PP

RCC_APB1PeriphClockCmd(RCC_APB1Periph_TIM2 , ENABLE);

TIM_InternalClockConfig(TIM2);

TIM_TimeBaseInitTypeDef ...
//用你屁股想一下，我们Counter和分频分别写100和720可以获得100的分辨率，也就是分为100份数
//而这100份是被平均分配在1/1000秒内的，所以CCR设50就可以获得50%的占空比
TIM_TimeBaseInit(TIM2 , &...)

//接下来是OC的初始化

TIM_OCInitTypeDef TIM_OCInitStructure;
TIM_OCStructInit(&TIM_OCInitStructure); //因为有很多通用计时器用不到的，所以这里都给个默认值，不然值不确定
TIM_OCInitStructure.TIM_OCMode = TIM_OCMode_PWM1; //输出比较模式
TIM_OCInitStructure.TIM_OCPolarity = TIM_OCPolarity_High;
TIM_OCInitStructure.TIM_OutputState = TIM_OutputState_Enable;
TIM_OCInitStructure.TIM_Pulse = ; //设置CCR (16位) 的值
TIM_OC1Init(TIM2 , &TIM_OCInitStructure)
```

**这里拓展一下引脚重映射的方法**
因为TIM2有可能和其他复用引脚功能占用同一个口，为了解决这个问题，可以把TIMCHx口重映射到其他口上，通过AFIO来实现

```C
RCC_APB2PeriphClockCmd(RCC_APB2Periph_AFIO,ENABLE);  
GPIO_PinRemapConfig(GPIO_PartialRemap1_TIM2 , ENABLE);
//这里注意一一下有三种模式
//PartialRemap1是吧OC1,OC2重映射到PA14和PA15上
//PartialRemap2是重映射OC3和OC4
//FullRemap就是全部重映射了

//还要注意，PA15默认为JTDI的调试端口，我们需要取消这种模式
GPIO_PinRemapConfig(GPIO_Remap_SWJ_JTAGDisable,ENABLE);
```

#### 驱动舵机
舵机要求50Hz的频率，那么PSC设置72，ARR设置20K即可
这时，CRR的范围是500~2500，我们可以根据这个公式来算角度需要的CCR值
$$
CCR = \left({\alpha\over180\degree}\cdot2000\right) + 500
$$
然后去把那个按钮判断的模块偷过来就行了
